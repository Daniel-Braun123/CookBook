01 — Projektübersicht

Was ist CookBook?

Eine moderne Webanwendung zum Erstellen, Durchsuchen, Bewerten und Speichern von Rezepten — mit lokalem Login und Google OAuth2.

  • Rezeptverwaltung — CRUD mit Zutaten, Kochschritten & Nährwerten
  • Bewertungssystem — 1–5 Sterne mit aggregiertem Rating
  • Volltextsuche — Titel & Beschreibung
  • Kategoriefilter — Rezepte nach Kategorie
  • Favoriten — Rezepte speichern / unspeichern
  • Admin-Panel — Benutzerverwaltung & Moderation

07 — Beispiel: Rezept erstellen

POST /api/recipes/create

Request Body

{
  "title": "Spaghetti Carbonara",
  "description": "Klassisch italienisch...",
  "image": "https://res.cloudinary.com/.../img.jpg",
  "categoryName": "Pasta",
  "difficulty": "MITTEL",
  "prepTime": 15,
  "cookTime": 20,
  "servings": 4,
  "tags": ["italienisch", "schnell"],
  "ingredients": [
    { "name": "Spaghetti", "amount": 400, "unit": "g" },
    { "name": "Guanciale", "amount": 200, "unit": "g" }
  ],
  "steps": [
    { "instruction": "Wasser kochen", "duration": 10 },
    { "instruction": "Guanciale braten", "duration": 5 }
  ],
  "nutrition": {
    "calories": 580, "protein": 22,
    "carbs": 65, "fat": 24
  }
}

Datenfluss im Backend

@PostMapping("/create")
public RecipeResponseDto create(
    @RequestBody CreateRecipeRequestDto req,
    @AuthenticationPrincipal Users user
) {
    return recipeService.createRecipe(req, user);
}

// RecipeService.createRecipe():
→ categorieRepo.findByName(name)
→ recipeRepo.save(recipe)
→ ingredientsRepo.saveAll(ingredients)
→ cookingStepsRepo.saveAll(steps)
→ nutritionInfoRepo.save(nutrition)
→ RecipeResponseDto.fromEntity(recipe)

Response

  "id": 42,

  "rating": 0.0,
  "reviewCount": 0,
  "author": { "id": 1, "name": "Max" },
  "category": { "name": "Pasta" },
  // ... alle Felder
}

11 — Frontend

Frontend — Kurzübersicht

Das Frontend wurde größtenteils mit KI-Unterstützung erstellt und dient als Single-Page-Application für die Interaktion mit der REST API.

Technologie

Angular 17 (Standalone Components) · Tailwind CSS · Angular Material

Komponenten

25+ Komponenten — Rezepte, Auth, Profil, Admin, Suche, UI-Elemente

API-Kommunikation

12 Services nutzen HttpClient · Auth-Interceptor fügt JWT hinzu · Error-Interceptor handhabt 401

Routing

Öffentliche Routen (Home, Rezeptdetail, Suche), geschützte Routen mit Guards (Profil, Create, Saved, Admin)

Guards

authGuard · adminGuard · guestGuard · pendingChangesGuard

Bild-Upload

Direkt zu Cloudinary — 5MB Limit, Frontend-seitige Validierung

Hinweis: Der Fokus dieser Präsentation liegt auf dem Backend. Das Frontend wurde überwiegend mit KI-Tools generiert und wird hier nur der Vollständigkeit halber erwähnt.
