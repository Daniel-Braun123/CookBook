@PostMapping("/create")
public RecipeResponseDto create(
    @RequestBody CreateRecipeRequestDto req,
    @AuthenticationPrincipal Users user
) {
    return recipeService.createRecipe(req, user);
}

08 — Code-Beispiel: GET

GET /api/recipes/getRecipeById

Controller

@GetMapping("/getRecipeById")
public RecipeResponseDto getRecipeById(
    @RequestParam String recipeId
) {
    return recipeService.toResponse(
        recipeRepository.getRecipeById(recipeId)
    );
}

Aufruf vom Frontend

// Beispiel-URL:
GET /api/recipes/getRecipeById?recipeId=42

Was passiert hier?

@GetMapping

Spring sagt: „Dieser Endpoint reagiert auf HTTP-GET unter /getRecipeById."

@RequestParam

Liest den Wert recipeId aus der URL und gibt ihn als String in die Methode.

Repository

Holt das Rezept aus der Datenbank — die SQL erzeugt Spring automatisch.

DTO als Response

Die Entity wird in ein RecipeResponseDto gemappt und als JSON zurück an den Client gesendet.

09 — Umsetzung: Security

JWT + OAuth2 Authentifizierung

Lokaler Login (JWT)

1. POST /api/auth/login

Email + Passwort an Backend

2. BCrypt.matches()

Passwort gegen Hash vergleichen

3. JWT signieren

HMAC-SHA256 · 24h Gültigkeit · enthält userId + role

4. Token an Client

Frontend speichert Token im LocalStorage

5. Folgende Requests

Authorization: Bearer ... → JwtAuthFilter → SecurityContext

Security-Schichten

BCrypt — Passwort-Hashing mit Salt
JWT — Stateless · 24h · HMAC-SHA256
@PreAuthorize — Method-Level Security
Rollen — USER · ADMIN (in JWT enthalten)
10 — Aufgetretene Probleme

Herausforderungen & Lösungen

🔑 Spring Boot Projekt aufsetzen

Problem: Setup mit Gradle, PostgreSQL & Dependencies.

→ Lösung: Doku gelesen, Schritt für Schritt lauffähig gemacht.

🌐 Datenbank verbinden

Problem: URL, User, Passwort & Treiber mussten stimmen.

→ Lösung: application.properties geprüft, Fehler ausgewertet.

⚡ Tabellen richtig verknüpfen

Problem: JPA-Beziehungen erstmal verstehen.

→ Lösung: Mit kleinen Test-Entities geübt.

⭐ REST-API testen

Problem: Requests & Responses mussten zum Frontend passen.

→ Lösung: Mit Postman getestet, DTOs angepasst.

🗄 Validierung von Eingaben

Problem: Leere oder fehlerhafte Daten führten zu Fehlern.

→ Lösung: Pflichtfelder definiert, Eingaben geprüft.

🖼 Bilder-Upload

Problem: Eigene Speicherung wäre zu aufwendig.

→ Lösung: Cloudinary als externer CDN, nur URL speichern.

11 — Lessons Learned

Was ich gelernt habe

🏗 Architektur

  • Erst planen, dann coden — spart später viel Refactoring
  • Datenmodell zuerst entwerfen — das Fundament der ganzen App
  • Klare Ordnerstruktur hilft — Dateien schneller wiederfinden

🔧 Praxis

  • Flyway von Anfang an — kein manuelles Schema-Management
  • Security früh konzipieren — nachträglich ist es schmerzhaft
  • Konsistente API-Pfade sparen Doku

💡 Mindset

  • Funktional first — Performance kommt später
  • Aussagekräftige Commits = nachträgliche Doku
  • Iterativ statt perfekt — kleine, testbare Schritte
